# Material Writing — 资料驱动深度写作

从「我有一份资料」到「完整深度文章」的全链路写作系统。

**核心理念**：以人为主体——AI 提问引导，用户深化思考，最终输出的是用户的思考而非 AI 代写。

## 适用场景

- "基于这份资料写一篇文章"
- "帮我深入分析这篇文章"
- 用户上传/引用资料，想要深度思考和写作
- 批量处理多篇资料的写作项目

## 与原系统的关系

本 Skill 从 [material-writing-system v1.2.0](https://github.com/Tasihi89/vibe-writing-workflow) 迁移而来，核心改编：

| 原系统 | remio 适配 |
|--------|-----------|
| Claude Code CLI + 本地文件 | remio Agent + 知识库 Note |
| Obsidian 双链 `[[]]` | remio `:remio-inlink[]` |
| 本地 `项目/` 目录 | Collection + Notes |
| `project.yaml` 状态文件 | 项目状态笔记 |
| Obsidian Canvas 白板 | Collection 排序 |
| `pwd` 检测工作目录 | 知识库上下文自动获取 |

## 工作流总览

```
Phase 0 项目初始化 →
Phase 1 资料解析 → 资料卡片 →
Phase 2 提问生成 → 探索方向 →
Phase 3 思考扩展 → 苏格拉底式对话 → 扩展卡片 →
Phase 4 结构规划 → 输出卡片 →
Phase 5 迭代优化 → 润色 →
Phase 6 最终成稿
```

### 卡片层级

```
资料卡片（保留论证肌理，600-1800字）
  → 回答卡片（用户原始回答，逐字保留）
    → 扩展卡片（资料+对话提炼，比资料卡片更深刻）
      → 输出卡片（面向读者的文章化表达）
        → 最终成稿（串联所有输出卡片，消除卡片感）
```

## 核心路由：意图识别

收到用户消息后，按以下逻辑路由：

### 新项目

**触发**：用户上传资料 / 引用笔记 / 说"基于XX写文章" / "分析这份资料"

**执行**：
1. 进入 Phase 0（项目初始化）
2. 创建项目 Collection + 状态笔记
3. 进入 Phase 1（资料解析）

### 恢复项目

**触发**：用户说"继续" / "接着写" / 指定已有项目名

**执行**：
1. `search_notes` 搜索项目状态笔记（标题含「📋 项目状态」）
2. 读取状态，判断当前阶段
3. 加载对应 Phase 继续

### 列项目

**触发**：用户说"有哪些项目" / "列出项目"

**执行**：
1. `search_notes` 搜索所有项目状态笔记
2. 列出项目名、阶段、更新时间

### 快捷命令

**触发**：用户输入 `/结构`、`/迭代` 等

**执行**：直接跳转到对应 Phase，无需确认

### 普通对话

**触发**：不涉及写作工作流的对话

**执行**：正常回答，不加载项目状态

## Phase 0: 项目初始化

### 执行步骤

1. **提取项目名**：从资料标题或用户描述中提取
2. **创建 Collection**：`📝 [项目名] · 写作项目`
3. **创建项目状态笔记**：`📋 [项目名] · 项目状态`

### 项目状态笔记格式

```markdown
# 📋 [项目名] · 项目状态

**项目名称**：[项目名]
**创建时间**：[YYYY-MM-DD]
**模式**：资料驱动

## 📊 项目进度
**当前阶段**：资料解析
**整体进度**：5%

## 📚 资料状态
**资料数量**：0份
**已解析**：0份
**待解析**：0份

## 📄 资料列表
（暂无）

## 📋 资料卡片
（暂无）

## 💡 扩展状态
**当前轮次**：第0轮
**距上次生成**：0轮
**已生成扩展卡片**：0张

## 📝 已生成的扩展卡片
（暂无）

## 📋 待转化扩展卡片
（暂无）

## ✨ 输出状态
**当前正在优化**：无

## ✨ 输出卡片（优化后）
（暂无）

## 🔍 待探索问题
**未完成**：0个
```

4. **创建待探索问题笔记**：`❓ [项目名] · 待探索问题`
5. 将状态笔记和待探索问题笔记加入 Collection

## Phase 1: 资料解析

**读取 [material-parser.md](./references/material-parser.md)** 获取完整工作流。

### 核心输出
- 资料卡片（每张 600-1800 字，保留论证肌理）
- 四维度问题识别（空白/矛盾/延伸/对比）
- 初始化待探索问题池

### 关键原则
- **资料卡片 ≠ 摘要**：保留论证路径、关键例子、类比、原文口吻
- **资料肌理优先**：判断一段内容是否保留，看它在原资料中的功能
- **宁可稍长，不要抽到只剩概念骨架**

### 完成后
自动提示进入 Phase 2（提问生成），或让用户选择感兴趣的资料卡片直接进入 Phase 3。

---

## Phase 2: 提问生成

**读取 [question-generator.md](./references/question-generator.md)** 获取完整工作流。

### 核心输出
- 2-4 个探索方向（每个方向 2-3 个具体问题）
- 探索方向列表笔记
- 同步待探索问题

### 问题评分公式
`总分 = 深度值(1-5) × 2 + 相关性(1-5) × 1.5 + 探索价值(1-5) × 1.5`

### 完成后
用户选择方向 → 进入 Phase 3（思考扩展）

---

## Phase 3: 思考扩展（苏格拉底式对话）

**读取 [thinking-expander.md](./references/thinking-expander.md)** 获取完整工作流。

### 核心机制
- **AI 提问，用户思考**（不是 AI 回答）
- 每 4 轮自动生成扩展卡片
- 每轮保存完整回答卡片（用户原话 + AI 回应）

### 提问策略
1. 第 1 轮：从用户直觉出发
2. 第 2 轮：引导对比思考
3. 第 3 轮：深入底层原理
4. 第 4 轮：回归核心问题，形成闭环

### 关键原则
- **用户原话逐字保留**：不总结、不压缩、不改写
- **每轮保存后回读审核**
- **第 4 轮必须自动生成扩展卡片**，不得只提示
- 每轮更新待探索问题（标记已回答 + 追加新问题）

### 完成后
提示用户：继续探索 / 进入结构化（`/结构`）

---

## Phase 4: 结构规划

**读取 [structure-planner.md](./references/structure-planner.md)** 获取完整工作流。

### 核心输出
- 输出卡片（面向读者的文章化表达）
- 叙事边界审计（红线/诚实线/绿区）

### 关键原则
- **输出卡片必须比扩展卡片更好**
- **认知轨迹**：按人的真实思考过程组织，不是教科书逻辑
- **叙事边界审计**：生成前区分红线/诚实线/绿区，防止过度主张
- 回看回答卡片，保留用户原始思考

### 完成后
提示用户：迭代当前卡片 / 继续结构化 / 进入成稿

---

## Phase 5: 迭代优化 & 润色

**读取 [iterative-writer.md](./references/iterative-writer.md)** 获取完整工作流。

### 润色三原则（不可违反）

1. **用户框架神圣不可侵犯**：删掉的内容永不恢复
2. **只优化表达，不改变内容**
3. **每处改动问：是让原有内容更清晰，还是添加了新内容？**

### 事实核查
涉及数据/时间/技术规格时，自动 `web_search` 验证。

### 中文写作风格
**读取 [style-guide.md](./references/style-guide.md)** 获取完整规范。

核心：写的是中文，不是翻译的英文。起承转合螺旋、留白段落、问句过渡。

### 完成后
提示用户：继续迭代 / 进入成稿（`/成稿`）

---

## Phase 6: 最终成稿

**读取 [final-draft.md](./references/final-draft.md)** 获取完整工作流。

### 核心输出
- 最终成稿笔记
- 质量检查报告

### 成稿前检查
1. 确认「当前正在优化」为「无」
2. 确认待探索问题已处理（未完成问题需用户确认搁置）
3. 涉及事实/数据时加载信息核查

### 成稿核心规则
- 保留逻辑，大胆优化表达
- 消除卡片感
- 制造认知张力（基于前文的真实疑问，不靠修辞）

### 完成后
更新项目状态为 `done`，同步 Collection。

## 质量规则

1. **全程留痕**：每个决策都记录在笔记中
2. **人的主体性**：AI 识别问题但用户选方向，AI 提方案但用户做决定
3. **回答卡片逐字保留**：不总结、不压缩、不改写
4. **问题闭环**：待探索问题从资料解析追踪到成稿前
5. **润色不恢复**：用户删掉的内容永远不恢复
6. **资料肌理优先**：宁可稍长，不要只剩概念骨架
7. **所有笔记加入 Collection**：每创建一个笔记就加入项目 Collection
