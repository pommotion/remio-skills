# Phase 6: 最终成稿 Agent

将输出卡片串联成完整文章，消除卡片感，制造认知张力。

---

## 工作流

### 阶段 1：读取输出卡片

1. `search_notes` 搜索项目 Collection 中的输出卡片
2. 按项目状态笔记中的顺序列出
3. 用户确认顺序

### 阶段 1.5：成稿前风险检查

1. `read` 项目状态笔记，确认「当前正在优化」为「无」
2. `read` 待探索问题笔记
3. 如仍有未完成 `[ ]`，提示用户确认：
   - 继续探索后再成稿
   - 或有意搁置，继续成稿
4. 涉及事实/数据时，`web_search` 核查

### 阶段 2：读取卡片内容

`read` 依次读取所有输出卡片完整内容。

### 阶段 3：生成最终成稿

`create_note` 创建，标题：`📝 [项目名] · 最终成稿`

**核心原则**：保留逻辑，大胆优化表达，制造认知张力。

**必须做到**：
- 保留论证链条（大/小逻辑不变）
- 大胆优化表达（悬念、冲突、追问）
- 拆分长句、删除啰嗦、突出重点
- 消除卡片感

**禁止**：
- 复制原文
- 改变核心逻辑
- 自作主张添加新论点
- 生硬分隔

**成稿六规则**：

1. **开局**：反直觉问题/现象，制造好奇心
2. **中间**：层层递进，追问式推进（现象 → 为什么 → 原理 → 本质）
3. **结尾**：回到开头，给出本质答案
4. **过渡**：融入内容改写，不用简单过渡词
5. **语言**：对话感（用"你"、"我们"、"想想看"），设问推进
6. **真实困惑**：保留惊讶（"太诡异了！"、"凭啥？！"）

**执行策略**：逐张卡片优化，边读边写。

### 阶段 4：质量检查

- 逻辑连贯性
- 无卡片感
- 无 AI 腔
- 叙事边界合规（红线内容语气正确，反面证据已展示）
- 中文风格自然

- 当前阶段：`done`
- 整体进度：100%
- 最终成稿路径和生成时间
