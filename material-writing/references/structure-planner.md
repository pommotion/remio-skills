# Phase 4: 结构规划 Agent

理解扩展卡片 → 拆成输出卡片 → 用认知逻辑和叙事手法写出更好的首版。

---

## 工作流 1：分析下一步写什么

**触发**：`/结构`、"结构化"、"输出"

### 1. 读取项目全貌

- `read` 项目状态笔记
- `read` 待探索问题笔记
- `search_notes` 获取待转化扩展卡片和已有输出卡片

### 2. 待探索问题风险检查

如果仍有未完成 `[ ]`：
```
还有 X 个待探索问题未完成。现在可以：
1. 先继续探索这些问题
2. 暂时搁置，继续结构化
```

### 3. 分析核心问题

- 这个项目最强的中心矛盾是什么？
- 已完成输出卡片建立了什么基础？
- 还没转化的扩展卡片里，哪张最适合作为下一段认知递进？

### 4. 提供方案

```
我看了当前项目结构，有三个可走方向：

方案A：接下来写《[扩展卡片标题]》
- 位置：承接[已有输出]
- 价值：把[核心矛盾]推进到[下一层]

我的推荐：方案X，理由是[...]

你选哪个？
```

---

## 工作流 2：价值点分析 + 生成首版

### 1. 读取输入

必须读取：当前扩展卡片、已有输出卡片
按需读取：相关资料卡片、回答卡片

### 2. 价值点分析（先给方案再动笔）

```
我分析了《[扩展卡片标题]》，它适合拆成 X 个输出节点：

1. [节点1]：承接上游，抛出[困惑/矛盾]
2. [节点2]：解释[核心机制/关键对比]
3. [节点3]：回到用户洞察，留下[新问题]

确认按这个结构生成吗？
```

### 3. 叙事边界审计（生成前必做）

对每张即将生成的输出卡片做边界审计：

```markdown
## 叙事边界清单

### ❌ 红线（不能主张的）
- [推测、过度概括、证据不足的判断]

### ⚠️ 诚实线（必须在正文展示的反面证据）
- [从回答卡片和待探索问题中提取]

### ✅ 绿区（可安全主张的）
- [有多轮对话验证、有资料支撑的论点]
```

### 4. 生成首版输出卡片

`create_note` 创建，标题：`✨ [项目名] · 输出：[节点标题]`

**生成要求**：
- 从读者困惑进入，不从资料定义进入
- 把用户洞察写成可读文章，不保留问答腔
- 保留有价值的原始例子、类比、情绪和反直觉表达
- 补足必要上下文，不添加未经来源支撑的新论点
- 无一级标题，内容从二级标题开始
- 无空行（紧致排版）

### 5. 回读检查

- 无一级标题
- 无空行
- 无来源引用块
- 红线内容用了"也许""一种可能"等语气
- 诚实线中的反面证据在正文中展示

### 6. 更新项目状态

- 输出卡片列表
- 待转化扩展卡片（移除已转化的）
- 文章结构分析

---

## 核心原则

1. **输出卡片必须比扩展卡片更好**：深度读取扩展卡片，必要时回看回答卡片
2. **认知轨迹优先**：按人的真实思考过程组织，不是教科书逻辑
3. **叙事边界审计**：防止过度主张
4. **从困惑开始**：不从定义开始
5. **遇到卡点就打断**：不延后解释
