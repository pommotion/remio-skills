# Phase 1: 资料解析 Agent

将原始资料解析为保留论证肌理的资料卡片，识别四维度问题。

---

## 工作流

### 步骤 1：读取资料

- 如果用户引用了知识库笔记：`read note://[noteId]` 读取全文
- 如果用户上传了文件：`read` 读取文件内容
- 如果用户粘贴了内容：直接使用
- 如果用户给了 URL：`web_fetch` 抓取内容

**支持来源优先级**：知识库笔记 > 本地文件 > URL > 粘贴内容

### 步骤 2：智能分段

**分段策略**：
1. 检测结构标记（Markdown 标题 → 按章节；空行 → 按段落；列表 → 按逻辑单元）
2. 每段至少包含一个完整观点，避免 mid-sentence 切分
3. 长度控制：600-1800 字/卡片；过短合并，过长拆分但不拆散同一论证单元

**资料过长（>5000 字）**：
- 询问用户："资料较长，全文解析还是重点解析？"
- 全文解析：按完整论证单元切分，不为控制数量压掉关键例子
- 重点解析：列出章节让用户选择

**资料过短（<200 字）**：合并为一张卡片

### 步骤 3：逐段解析

对每个段落执行：

#### 3.1 提取核心观点（1-3 个）
- 识别主题句（段首或段尾）
- 提取关键论断/结论
- 保留承载论证的例子、类比、反问和转折
- 只压缩重复铺垫，不压缩帮助理解的叙事材料

#### 3.2 识别关键概念
- 专业术语、定义、专有名词
- 重要数据（数字、参数、时间点）

#### 3.3 四维度问题识别

| 维度 | 触发条件 | 示例 |
|------|---------|------|
| **空白点** | 提到但未展开 | "提到自注意力但没解释为什么比RNN快" |
| **矛盾点** | 前后逻辑冲突 | "说更快但复杂度O(n²)" |
| **延伸点** | 可深化的方向 | "能用到所有序列任务吗？" |
| **对比点** | 可与其他知识对比 | "和人类注意力有什么相似？" |

#### 3.4 提取原文肌理

必须记录：
- **论证路径**：起点 → 转折 → 例子 → 结论
- **关键例子**：作者用来解释观点的场景、故事、比喻
- **可复用表达**：短句、判断句、反问句、定义句
- **语气特征**：困惑、反直觉、递进、收束等

**判断标准**：
- 删掉这个例子，观点变抽象 → 不能删
- 删掉这句表达，未来成稿失去原文气质 → 保留
- 只是来源注释、长注释、重复铺垫 → 可压缩

### 步骤 4：生成资料卡片笔记

每张卡片一个 `create_note`，标题：`📖 [项目名] · 资料卡片：[主题]`

**卡片格式**：

```markdown
> **资料来源**：[来源标题]
> **原文锚点**："[关键引文]"

## 资料导读
[1-3句自然段说明：这段资料的重点不是XX，而是XX。后续问题应围绕XX展开。]

## 作者是怎么讲出来的
- [论证路径：先XX → 再XX → 最后XX]

## 可复用材料
- **例子**：[具体例子]
- **表达**：[可复用的精炼表达]

## 核心观点
- [观点1]
- [观点2]

## 关键概念
- **[术语]**：[一句话解释]

## 潜在问题
### 空白点
- [ ] [问题1]
### 矛盾点
- [ ] [问题2]
### 延伸点
- [ ] [问题3]
### 对比点
- [ ] [问题4]
```

### 步骤 5：更新项目状态

`update_note` 更新项目状态笔记：
- 资料状态：资料数量、已解析、待解析
- 资料卡片列表
- 进度：15%

### 步骤 6：同步待探索问题

将所有资料卡片中的潜在问题汇总到待探索问题笔记：
- 按资料卡片分组
- 保留问题类型
- 默认全部为 `[ ]`

### 步骤 7：完成提示

```
资料解析完成！生成了 X 张资料卡片，识别了 Y 个潜在问题。

接下来我可以帮你：
1. 查看具体的问题列表（说"有什么可以思考的？"或 /提问）
2. 直接选择感兴趣的资料卡片深入探讨

你想怎么做？
```

---

## 设计原则

1. **准确提取，不过度解读**：核心观点基于资料原文，不添加资料中没有的内容
2. **保留丰富度**：资料卡片是写作底稿，不是复习提纲
3. **问题质量优先于数量**：宁可少而精，不要多而泛
4. **为下一阶段做准备**：资料卡片是提问生成的基础
