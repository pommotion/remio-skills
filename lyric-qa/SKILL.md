# Lyric QA — 歌词质量检测器

输入一段歌词，从 4 个维度自动检测质量，给出评分和改进建议。

## 适用场景

- 用户写完歌词，想检查质量
- 用户说"帮我检查歌词" / "这段歌词好不好" / "lyric qa"
- 用户想让 AI 生成的歌词通过质量门径后再用
- 作为 `scene2lyric` 或 `ai-music-producer` 的质量检查环节

---

## 检测维度

### D1. 陈词滥调扫描 🔍

**规则来源**: 场景辞海·设定常见陷阱 (moojx616nj3lhs5j08p)

**扫描清单**：

| 陈词滥调类型 | 典型表达 | 替换建议 |
|-------------|---------|---------|
| 直接情绪 | "我好想你""心痛""心碎""难过" | 用场景/物件暗示情感 |
| 天气套路 | "暴雨如注""大雪纷飞"（暗示悲伤时） | 换晴天反差，或用微天气（细雨/雾气） |
| 身体套路 | "泪水模糊双眼""心如刀割""窒息" | 用反直觉的身体反应（"手指不自觉地摸了手机"） |
| 时间套路 | "永远""一生一世""天长地久" | 用具体时间单位（"到冰箱过期的那天"） |
| 动作套路 | "转身离去""独自走在雨中" | 换空间/动作细节 |
| 修辞套路 | "你是我的一切""失去你世界暗了" | 用对比/反差替代直接比喻 |

**评分标准**：

| 等级 | 标准 | 分数 |
|------|------|------|
| 🟢 优秀 | 无陈词滥调，每段用新鲜表达 | 90-100 |
| 🟡 良好 | 1-2 处轻微套路，可接受 | 70-89 |
| 🟠 需改进 | 3+ 处套路表达 | 40-69 |
| 🔴 重写 | 大量陈词滥调，缺乏原创表达 | 0-39 |

**输出格式**：

```
### D1. 陈词滥调扫描

评分：🟡 78/100

| 行号 | 原文 | 问题类型 | 建议 |
|------|------|---------|------|
| L3 | "泪水模糊了双眼" | 身体套路 | 用反直觉的身体反应替代 |
| L7 | "独自走在雨中" | 动作套路 | 换场景或换天气 |
```

### D2. 画面感评分 🎨

**核心指标**：具体物象 vs 抽象情感的比例。

**检测方法**：

1. 逐行扫描歌词
2. 将每行分类为：
   - **画面行**：含具体物象/场景/感官描写（"冰箱里还有你买的药"）
   - **情绪行**：含抽象情感词（"我好孤单"）
   - **叙事行**：含具体动作/事件（"你关上门"）
   - **留白行**：感叹词/语气词/重复句（"Oh~"）
3. 计算画面行占比

**评分标准**：

| 等级 | 画面行占比 | 分数 |
|------|----------|------|
| 🟢 优秀 | ≥ 70% | 90-100 |
| 🟡 良好 | 50-69% | 70-89 |
| 🟠 需改进 | 30-49% | 40-69 |
| 🔴 重写 | < 30% | 0-39 |

**五感覆盖检查**：

统计歌词中使用的感官种类：

| 感官 | 检测关键词示例 |
|------|--------------|
| 视觉 | 光/暗/色/影/形/远/近 |
| 听觉 | 声/音/响/静/嗡/滴答 |
| 嗅觉 | 香/味/臭/气息/味道 |
| 触觉 | 冷/热/软/硬/凉/温 |
| 味觉 | 苦/甜/酸/咸/涩 |

**覆盖 ≥ 3 种感官为合格。**

**输出格式**：

```
### D2. 画面感评分

评分：🟢 88/100

画面行分布：
  主歌A：███████░░░ 70% 画面为主
  主歌B：████████░░ 80% 叙事+画面
  副歌：  ██████░░░░ 60% Hook重复拉低比例（正常）
  桥段：  █████████░ 90% 高画面感

五感覆盖：视觉✅ 听觉✅ 嗅觉✅ 触觉✅ 味觉❌
（4/5 覆盖，优秀）
```

### D3. Hook 强度 🪝

**规则来源**: AI 音乐手册·Hook 铁律 + 爆款方法论·钩子公式

**检测项目**：

| 项目 | 标准 | 权重 |
|------|------|------|
| 长度 | 副歌核心句 ≤ 9 字 | 30% |
| 重复 | 核心句在副歌重复 ≥ 2 次 | 30% |
| 记忆点 | 含重复字或叠字 | 20% |
| 开头 | 第一个字是动词/感叹词/具体名词 | 20% |

**评分标准**：

| 等级 | 标准 | 分数 |
|------|------|------|
| 🟢 强 Hook | 4 项全通过 | 90-100 |
| 🟡 中等 | 3 项通过 | 60-89 |
| 🟠 弱 Hook | 2 项通过 | 30-59 |
| 🔴 无 Hook | ≤ 1 项通过 | 0-29 |

**输出格式**：

```
### D3. Hook 强度

评分：🟡 75/100

副歌核心句："冰箱里还有你买的药"
  ✅ 长度：9字（刚好达标）
  ✅ 重复：副歌出现3次
  ❌ 记忆点：无叠字/重复字
  ✅ 开头："冰箱"是具体名词

建议：考虑在核心句中加入重复字增强记忆点
  → "冰箱里还有你你你买的药"（太刻意）
  → 换思路：把核心句缩短到6-7字："冰箱里你的药"
```

### D4. 韵律一致性 🎵

**检测项目**：

| 项目 | 标准 |
|------|------|
| 主韵脚 | 全曲统一一个主韵脚（可换韵但不超过2次） |
| 段内一致性 | 同一段内韵脚不跳 |
| 节奏感 | 每行字数大致相近（允许±3字偏差） |
| 呼吸点 | 每行不超过15字（保证换气空间） |

**评分标准**：

| 等级 | 标准 | 分数 |
|------|------|------|
| 🟢 优秀 | 全曲统一韵脚，节奏均匀 | 90-100 |
| 🟡 良好 | 1次换韵，节奏基本均匀 | 70-89 |
| 🟠 需改进 | 2+次换韵或节奏不均 | 40-69 |
| 🔴 混乱 | 无韵脚规律，各行字数差异大 | 0-39 |

**输出格式**：

```
### D4. 韵律一致性

评分：🟢 92/100

韵脚分析：
  主歌A：药/了/好（ao 韵 ✅）
  主歌B：灯/冷/等（eng 韵 ✅，第1次换韵）
  副歌：  药/了/好（回到 ao 韵 ✅）
  桥段：  风/梦/中（eng 韵，与主歌B呼应 ✅）

节奏分析：
  平均行长：9.2字
  最长行：12字（副歌第3行）
  最短行：5字（副歌Hook）
  ✅ 无超长行
```

- D4 韵律一致性：15%

## 改进建议优先级

根据评分结果，按以下优先级给出建议：

1. **D1 < 70**：先清理陈词滥调——这是最基础的质量门径
2. **D2 < 70**：增加画面感——替换情绪行为画面行
3. **D3 < 60**：重写 Hook——副歌是歌曲的灵魂
4. **D4 < 70**：调整韵律——韵脚是歌词的音乐性基础

---

## 工作流

### 单次检测

用户给一段歌词时：

1. 读取完整歌词
2. 如果歌词引用了知识库笔记（`@[标题]{#noteId}`），用 `read` 获取完整内容
3. 按顺序执行 D1→D2→D3→D4
4. 输出综合评分报告
5. 给出按优先级排序的改进建议
6. **不主动修改歌词**——只给建议，修改由用户或用户触发的其他 skill 完成

### 与其他 skill 联动

- **scene2lyric**：生成歌词后自动触发 D2 画面感检测
- **ai-music-producer**：Phase 2 歌词阶段完成后触发完整四维检测

### 可选：参考场景辞海规则

当 D1 评分低于 70 时，可调用 `search_notes` 读取"设定常见的陷阱"(moojx616nj3lhs5j08p) 获取完整的反套路规则列表，给出更精准的替换建议。

## 质量规则

- 检测必须逐行扫描，不得跳过任何一行
- 评分必须给出具体行号和原文引用
- 建议必须是可操作的（"把 L3 改成..."而非"这里不够好"）
- 不改变用户的歌词内容——只给评分和建议
- 如果歌词不是中文，仍可检测 D2/D3/D4（D1 的陈词滥调清单仅适用于中文）
