# 作品资产上架指南

> 被 SKILL.md Phase 7 引用。上架前必读。
> ⚠️ 心态校准：不是「上传作业」，是「作品资产上架」。每首歌都是可复用的版权资产。

---

## 7.1 版权与来源记录（每首必存）

上传前必须确认并保存以下信息，这是版权争议、商业授权、平台审核时的核心凭证：

```
▎版权来源记录
  生成工具：MiniMax CLI / Suno
  模型版本：music-2.6 / ...
  生成日期：YYYY-MM-DD
  Prompt：[完整 Prompt 文本]
  歌词版本：V[N]
  是否二次修改：是/否
  音频文件名：[歌名]_v[N]_[日期].mp3
  封面生成工具：mmx image / Lovart / ...
  封面 Prompt：[完整封面 Prompt]
```

**禁止项**（版权红线）：
- ❌ 歌词抄袭已有歌曲、爆款改写、影视台词大段引用、名人语录、网络热梗原句堆叠
- ❌ 风格描述写「周杰伦风格」「像陈奕迅」「毛不易同款」→ 改为「华语独立民谣」「Urban Folk Pop」等流派名
- ❌ 封面使用明星脸、品牌 Logo、影视剧截图、他人摄影作品、版权字体

## 7.2 音频文件上传前检查

```
▎音频质检
  □ 文件能正常播放
  □ 文件名规范：[歌名]_v[N]_[日期].mp3（禁止 final_final_2.mp3 / test.mp3）
  □ 开头没有多余空白
  □ 结尾没有突然截断
  □ 人声没有明显爆音
  □ 音量不忽大忽小
  □ 中文断句自然
  □ 没有 AI 乱码词
  □ 副歌 Hook 能听清
  □ 歌曲不是未完成 demo 感
```

---

## 7.3 歌词-音频对照（必做）

AI 歌曲最常见的坑：**音频里唱错了，但提交的是原歌词。**

```
▎歌词-音频对照
  □ 实际唱出来的歌词和提交歌词一致
  □ 没有错字
  □ 没有敏感词
  □ 没有过度负面表达
  □ 没有品牌名、真实姓名、联系方式
  □ 没有大段英文或乱码
  □ 每段结构清楚（内容要分段）
```

如果唱错明显，**重新生成或修改歌词**，不要硬上传。

---

## 7.4 歌曲信息填写

### 歌名
- ✅ 简洁、可记、有情绪：`你有没有这样的时刻` / `只差一句` / `大人说明书`
- ❌ 不要：`AI原创歌曲001` / `情绪民谣测试版` / `Urban Folk Pop Demo`

### 歌手名 / 创作者名
提前固定一个统一名字，不要今天真名、明天网名、后天项目名。

### 制作信息
如实填写，不要虚构真实乐队或不存在的制作人：
```
  词：[用户] / AI 辅助
  曲：AI 生成 / [用户] 提示词设计
  编曲：AI 生成
  制作：[用户]
```

---

## 7.5 简介写「场景」不写「情绪」

模板：`一首写给[目标人群]的[风格]作品。[核心情感/主题]。适合[具体视频场景]。`

示例：
- ✅ `一首写给认真努力却暂时不被看见的人的 Urban Folk Pop。适合成长记录、深夜独处、Vlog 片尾和短视频情绪片段。`
- ❌ `一首伤感好听的歌，希望大家喜欢。`

---

## 7.6 标签三维度组合

| 类别 | 示例 |
|------|------|
| 情绪标签 | 孤独 / 治愈 / 怀旧 / 遗憾 / 成长 / 温暖 / 失眠 / 释然 |
| 风格标签 | 独立民谣 / Urban Folk Pop / Indie Pop / Dream Pop / Pop Jazz / 华语流行 / 情绪流行 |
| 场景标签 | 短视频 BGM / Vlog 片尾 / 城市夜归 / 成长记录 / 情绪短片 / 独处 / 回忆 |

**场景标签是视频创作者最有价值的差异化。**

## 7.7 AI 生成声明

如果平台有相关选项，**如实填写，不要隐瞒 AI 生成**。

通用声明：
```
本作品由本人完成主题策划、歌词创作/修改、风格设计、提示词设计与筛选，音乐生成环节使用 AI 工具辅助完成。
```

MiniMax 专用：
```
音乐生成工具：MiniMax CLI。本人负责选题、歌词、风格设定、提示词设计、版本筛选和上架资料整理。
```

## 7.8 封面生成

### 默认工具：bizyair-skill（GPT Image 2 via ModelZoo o2-t2i）

封面生成优先使用 bizyair-skill 的 modelzoo-run 命令（GPT Image 2 画质最佳）。

```bash
# 通过 bizyair-skill CLI 生成（始终异步，自带轮询，不会重复提交）
python3 ~/.remio/skills/bizyair-skill/scripts/cli.py modelzoo-run \
  bza-image-o2-base/text-to-image \
  --param "prompt=[封面描述，英文，包含：风格 + 主色调 + 核心元素]" \
  --api-key $BIZYAIR_API_KEY
```

**⚠️ 注意**：
- bizyair-skill 使用 ModelZoo API，始终异步提交（< 5s 返回 request_id），自带轮询
- 不会因同步超时导致重复提交（之前 bozo-aigc 同步模式的 bug 已修复）
- 需要 `BIZYAIR_API_KEY` 环境变量已设置
- 需要代理：`export HTTPS_PROXY=http://127.0.0.1:7890`（DNS 污染绕过）
- 比例固定 `1:1`（平台封面标准），在 prompt 中指定
- 批量生成推荐使用 `music-vault/regenerate_covers.py`（已适配 bizyair-skill）

### 回退方案：mmx image

当 bizyair-skill 不可用（`BIZYAIR_API_KEY` 未设置 / API 报错）时：

```bash
mmx image generate \
  --prompt "[封面描述，中文，包含：风格 + 主色调 + 核心元素 + '无文字，无水印']" \
  --ratio 1:1 \
  --out ~/Desktop/📂\ 音乐/[歌名]/cover_[歌名].png
```

### 封面描述示例
- 都市民谣 → 留白极简风格，温暖米色调，城市剪影，大面积纯色背景，手写歌名感，无文字无水印
- Dream Pop → 梦幻色调，朦胧光晕，极简几何元素，大面积留白，无文字无水印
- Pop Jazz → 复古暖色，咖啡厅氛围，单个极简乐器剪影，无文字无水印

**封面版权红线**：无明星脸、无品牌 Logo、无影视剧截图、无他人摄影、无版权字体、无平台水印、不模仿某张专辑封面。保存封面 Prompt 和生成日期。

**封面设计统一方案**：留白极简（Negative Space）— 大面积单色留白 + 一个极简意象 + 手写歌名，适合缩略图辨识、系列感强。

## 7.9 上架资料包输出格式

```
📦 作品资产上架包

▎歌曲信息
  推荐歌名：[歌名]
  副标题：[一句话定位]
  风格标签：[2-3个流派名]
  语言：中文
  作词：[用户名] + AI
  作曲：AI ([生成工具])

▎歌曲简介（100字内，含场景）
  [自动生成]

▎平台标签（三维度）
  情绪：[3-5个]
  场景：[3-5个]
  风格：[3-5个]

▎AI 生成声明
  [通用声明]

▎可剪辑片段
  15秒 Hook：[副歌起始时间码]
  30秒精华：[推荐截取区间]
  片尾余韵：[尾奏时间码]

▎推广文案（2-3版，带话题标签）
  文案 1：[核心角度]
  文案 2：[不同角度]

▎视频使用场景
  ✅ 最适合：[具体场景 + 原因]
  ✅ 也适合：[具体场景]
  ❌ 不适合：[具体场景 + 原因]
```

## 7.10 上传前总检清单（缺一不可）

```
▎上传前总检
  □ 版权来源记录已保存（工具/模型/日期/Prompt/封面Prompt）
  □ 音频质检通过（7.2）
  □ 歌词-音频对照通过（7.3）
  □ 歌名/歌手名/制作信息已规范填写（7.4）
  □ 简介含场景描述（7.5）
  □ 标签三维度齐全（7.6）
  □ AI 声明已准备（7.7）
  □ 封面已生成且无版权风险（7.8）
  □ 歌词没有明显唱错
  □ Hook 清晰可辨
  □ 已记录工具/日期/Prompt/版本号
```

## 7.12 平台适配

| 平台 | 格式 | 重点 | 注意事项 |
|------|------|------|----------|
| 网易云音乐 | WAV 44.1kHz | 封面 + 歌词 + 创作信息 | 音乐人认证流程 |
| 番茄音乐 | MP3/WAV | 完整版 + 简介 + 标签 + AI 声明 | 可能需 AI 生成声明 |
| 抖音/小红书 | MP3 15-30秒片段 | Hook 片段 + 话题标签 | 版权归属说明 |
| Spotify | WAV 44.1kHz 16bit | 英文元数据 + ISRC | 需发行商（DistroKid 等） |
