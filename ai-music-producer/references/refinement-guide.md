# 精修 + 诊断 + 后期处理指南

## Phase 5A：精修评估

### 六维诊断表

对生成的歌曲逐项打分（1-10），并给出具体问题和修改建议：

| 维度 | 听什么 | 打分标准 |
|------|--------|----------|
| **旋律** | 副歌是否抓耳？主歌副歌是否明显不同？听完能否哼出来？ | 10=一遍记住 / 7=几遍记住 / 4=记不住 |
| **人声** | 咬字是否清晰？有没有机械感？情绪是否到位？有无吞字/断句奇怪？ | 10=自然如真人 / 7=偶有AI感 / 4=明显AI |
| **编曲** | 有没有从少到多的动态？主歌是否留白？桥段是否收回来了？ | 10=完美动态 / 7=基本有变化 / 4=从头到尾一样 |
| **混音** | 人声是否被盖住？低频是否太轰？三设备（手机/耳机/音箱）是否一致？ | 10=平衡 / 7=偶有瑕疵 / 4=明显失衡 |
| **歌词** | 哪一句唱出来不自然？有没有吞字？画面感是否保留？ | 10=完美表达 / 7=偶有吞字 / 4=大量歌词被改 |
| **洗脑度** ⭐新增 | 副歌有没有重复结构？听完一次能否重复核心句？有没有「反差反打」？ | 10=立即上头 / 7=几遍记住 / 4=记不住核心句 |

**洗脑度评分细则**：
- 10分：副歌有明确重复结构（A+A+A+B/三字连发/签名式），且填入反套路意象
- 7-9分：有重复但不极致，或意象稍套路
- 4-6分：副歌无重复结构，旋律平
- 1-3分：整首歌无记忆点

### 视频适配评估

| 检查项 | 判断标准 |
|--------|----------|
| 15-30秒可截取？ | 副歌 Hook 能否独立成片段 |
| 人声是否抢画面？ | 如果做 BGM，人声是否过于突出 |
| 情绪是否匹配？ | 适合什么类型的视频（情绪短片/口播/Vlog/广告） |
| 开头3秒吸引力？ | 前奏是否能在3秒内抓住注意力 |
| 结尾是否干净？ | 是否有自然收尾点，还是突然断了 |

### 诊断报告输出格式

```
🎯 歌曲精修诊断报告

歌曲名称：[名称]
生成工具：Suno / Udio / 豆包 / MiniMax
总时长：[时长]

▎五维评分
  旋律：  [X/10] — [一句话评价]
  人声：  [X/10] — [一句话评价]
  编曲：  [X/10] — [一句话评价]
  混音：  [X/10] — [一句话评价]
  歌词：  [X/10] — [一句话评价]
  综合：  [XX/60]

▎视频适配
  适合用途：[片头 / 片尾 / 高潮铺底 / BGM / 主题曲]
  可截取片段：[具体时间码]
  15秒短视频：✅ 适合 / ❌ 不适合（原因）

▎主要问题
  1. [最严重的问题]
  2. [第二严重的问题]
  3. [第三严重的问题]

▎下一版 Prompt 修改建议
  - [具体修改项1]
  - [具体修改项2]

▎精修优先级
  1. [最需修的维度]
  2. [其次]
  3. [再次]
```

### 人声三问

每次诊断必须回答三个问题：
1. **哪一句咬字最不清？** → 标注时间码和歌词
2. **哪一句情绪最打动？** → 标注时间码和歌词
3. **哪一句听起来最像 AI？** → 标注时间码和歌词

## Phase 4B：自动精修循环（mmx 路径专属）

### 质量标准

| 总分 | 等级 | 自动操作 |
|------|------|----------|
| **≥ 48/60** | S | ✅ 达标，停止精修，提交用户审核 |
| **42-47** | A | ✅ 达标，停止精修，提交用户审核 |
| **36-41** | B | 🔄 自动进入精修循环 |
| **< 36** | C | 🔄 自动进入精修循环 |

**达标线 = 42/60（A 级）**。达到 A 级即停止。

### 精修循环流程

```
mmx generate → ASR 转录 → 五维诊断
    ↓
  总分 ≥ 35？
    ├── 是 → 停止，输出诊断报告
    └── 否 → 自动生成下一版（最小干预修改）
              ↓
          重新诊断 → 循环
```

### 最小干预原则

**先保亮点，再修问题**——绝不能为了修问题而丢掉上一版的亮点。

每次精修时，必须先标注上一版的亮点和问题：

```
V[N-1] 亮点（必须保留）：
  - [亮点1，如：男女和声]
  - [亮点2，如：Bridge 情绪爆发]

V[N-1] 问题（需要修复）：
  - [问题1，如：Outro 丢失]
  - [问题2，如：吞字]

V[N] 修改策略（最小干预）：
  - 修问题1 → [具体改法，不影响亮点的最小改动]
  - 修问题2 → [具体改法]
  - 保持亮点 → [明确写进 prompt 确保保留]
```

### 精修禁忌

1. ❌ 不要把所有问题一次性全修——每次最多修 2-3 个问题
2. ❌ 不要在 `--prompt` 中塞满修正指令——描述「想要什么」而非「不想要什么」
3. ❌ 不要强制加 `--structure`——除非上一版确实跳了关键段落
4. ❌ 不要把长句全拆短句——保留原始韵律，吞字用其他方式解决
5. ❌ 不要在 `--vocals` 中过度约束——保留上一版有特色的人声处理
6. ❌ 不要丢掉上一版的版本特点（如和声、特殊编曲）

**每次精修只做最小改动，验证效果后再决定下一步。**

### 精修上限

- 最多自动循环 **3 轮**（V1→V2→V3→V4）
- 3 轮后仍未达标 → 停止，输出所有版本诊断报告，由用户决定
- 每轮消耗 1 次 music-2.6 配额，循环前检查配额

### 版本留痕

每一轮必须记录：
1. **提示词变化**：V[N] vs V[N-1]，具体改了哪些参数
2. **诊断结果**：五维评分 + 亮点 + 问题
3. **修改策略**：基于什么诊断决定怎么改
4. **实际效果**：改了之后结果是变好还是变差

## Phase 5B：后期处理

### mmx Cover 精修（推荐）

```bash
# 基于初版音频换风格
mmx music cover \
  --prompt "[目标风格，如：民谣，木吉他，温暖男声]" \
  --audio-file [初版文件].mp3 \
  --out [歌名]_cover.mp3

# 指定歌词翻唱（修正咬字/断句问题）
mmx music cover \
  --prompt "[风格]" \
  --audio-file [初版文件].mp3 \
  --lyrics-file [修正后的歌词].txt \
  --out [歌名]_fixed.mp3
```

每次 cover 消耗 1 次 music-cover 配额（100 次/日）。

### mmx 重新生成（换人声/换风格）

```bash
# 换人声音色
mmx music generate \
  --prompt "[风格]" \
  --lyrics "[歌词]" \
  --vocals "[新人声描述]" \
  --model music-2.6 \
  --out [歌名]_v3.mp3

# 换编曲风格
mmx music generate \
  --prompt "[新风格]" \
  --lyrics "[歌词]" \
  --vocals "[人声]" \
  --instruments "[新乐器组合]" \
  --model music-2.6 \
  --out [歌名]_v4.mp3
```

### RVC 修音（可选）

```
RVC 修音建议：
1. 模型选择：[根据风格推荐 RVC 模型]
2. 音高修正范围：[建议参数]
3. 混响参数：[建议参数]
4. 参考：知识库中的 RVC 相关教程笔记
```

### 母带处理（推荐）

```
基础母带建议（Ozone / 在线工具）：
1. EQ：削减 200-300Hz（去除浑浊） + 提升 8-12kHz（增加空气感）
2. 压缩：ratio 2:1-4:1，阈值 -15dB 左右
3. 响度：目标 LUFS -14（流媒体）/ -16（短视频）
4. 限制器：防止削波

如果不想折腾：
- 用 LANDR / eMastered 在线母带服务
- 或直接在 Suno 的 "Enhance" 模式重新生成
```

### 导出规格

```
导出格式建议：
  短视频 BGM：MP3 320kbps / 15-30秒片段
  流媒体发布：WAV 44.1kHz 16bit / 完整版
  广告配乐：WAV 48kHz 24bit / 完整版 + 30秒版 + 15秒版
```
