# Scene2Lyric — 场景→画面感歌词生成器

输入情感关键词，从知识库的场景辞海中检索匹配场景，提取五感素材，替换抽象情感词，输出可直接喂给 Suno/Udio/豆包的画面感歌词。

## 适用场景

- 用户说"帮我写歌词" / "帮我写一首 XX 歌"
- 用户指定了情感（失恋/思念/励志/甜蜜/孤独/愤怒）但不知道怎么写出画面感
- 用户想用场景辞海的素材库来增强歌词的画面感
- 用户说"用 scene2lyric"

## 核心原理

AI 音乐手册的「画面感铁律」：**具体物象 > 抽象情感**。但手册没有给出系统化的画面感来源。

本 skill 填补这个缺口：用场景辞海的 222 个场景（田园 100+ / 都会 122）的五感描写作为素材库，把「写歌词」变成「查表 + 组装」。

**转换公式**：
```
1. 确定歌曲情感
2. 从场景辞海检索匹配场景
3. 提取五感素材（视觉+听觉为主，嗅觉+触觉+味觉为辅）
4. 用具体物象替换抽象情感词
5. 反套路检查（陷阱卡片）
6. 组装为结构化歌词
```

## 工作流

### Phase 1: 解析用户输入

从用户消息中提取：

| 信息 | 说明 | 默认值 |
|------|------|--------|
| **情感** | 核心情绪（失恋/思念/励志/甜蜜/孤独/愤怒/遗憾/释然） | 必填，缺失则询问 |
| **风格** | 音乐流派（流行/国风/R&B/嘻哈/摇滚） | 流行 |
| **主题** | 具体主题或故事（可选） | — |
| **目标工具** | Suno / Udio / 豆包 | Suno |

### Phase 2: 场景检索

用 `search_notes` 在场景辞海知识库中搜索匹配场景。

**情感→场景类型映射表**：

| 情感 | 推荐场景类型 | 搜索关键词 |
|------|-------------|-----------|
| 失恋 | 室内、私密空间 | 厨房 卧室 深夜 雨天 |
| 思念 | 车站、港口、老地方 | 车站 老街 港口 雨天 |
| 励志 | 天台、山顶、海边 | 天台 山顶 海边 日出 |
| 甜蜜 | 咖啡馆、公园、厨房 | 咖啡馆 公园 厨房 花园 |
| 孤独 | 深夜、空旷场所 | 深夜 便利店 天台 空旷 |
| 愤怒 | 暴雨、风暴、废墟 | 暴雨 风暴 雷 废墟 |
| 遗憾 | 老照片、旧物、秋天 | 秋天 老照片 旧物 书信 |
| 释然 | 晴天、海边、清晨 | 清晨 海边 晴天 微风 |

**搜索策略**：

1. 用情感对应的关键词调用 `search_notes`，`folder` = "场景设定创意辞典"
2. 取 3-5 个最相关的场景
3. 读取场景的完整五感描写内容
4. 如果搜索结果不足，扩大搜索范围或用 `rag` 补充

### Phase 3: 五感素材提取

从检索到的场景中，按优先级提取五感素材：

| 感官 | 优先级 | 歌词用途 | 提取方式 |
|------|--------|---------|---------|
| **视觉** | ⭐⭐⭐ | 画面建立 | 场景中的光影、色彩、形状描写 |
| **听觉** | ⭐⭐⭐ | 氛围营造 | 环境音、物体声、人声描写 |
| **嗅觉** | ⭐⭐ | 记忆触发 | 气味描写（最容易唤起记忆的感官） |
| **触觉** | ⭐⭐ | 情感暗示 | 温度、质感、身体感受 |
| **味觉** | ⭐ | 情感隐喻 | 苦/甜/酸作为情感隐喻 |

**提取原则**：

- 每个场景提取 2-3 个最强感官素材
- 优先选择「意外」的细节（不是人人都能想到的）
- 用名词短语而非完整句子（留空间给歌词韵律）

### Phase 4: 情感词替换

**替换规则**（核心转换）：

| 抽象情感词 | 替换方式 | 示例 |
|-----------|---------|------|
| "我好想你" | 用物件/场景暗示 | "冰箱里还有你买的药" |
| "你走了" | 用空间变化暗示 | "门口的拖鞋只剩一双" |
| "我好孤单" | 用环境氛围暗示 | "便利店的冷光白得刺眼" |
| "我爱你" | 用行为暗示 | "把你的名字写成密码" |
| "我很伤心" | 用身体感受暗示 | "咖啡凉了也没注意" |
| "永远" | 用具体时间替代 | "到冰箱过期的那天" |

**反套路检查**（基于场景辞海·设定常见陷阱）：

调用 `search_notes` 读取"设定常见的陷阱"(moojx616nj3lhs5j08p)，逐条检查歌词中是否出现陈词滥调：

- ✗ "泪水模糊双眼" → 找一个反直觉的身体反应替代
- ✗ "暴雨象征悲伤" → 换天气或用晴天反差
- ✗ "心如刀割" → 用具体的触觉描写替代
- ✗ "雨中独行" → 换场景或换天气

### Phase 5: 歌词组装

根据用户指定的风格，选择对应的结构模板组装歌词。

**默认结构（流行 6 段式）**：

```
主歌A（铺垫）：用场景的五感描写建立画面
主歌B（推进）：用物件/动作引入叙事
预副歌（蓄力）：情绪转折，节奏加快
副歌（爆发）：核心 Hook + 画面余韵
桥段（转折）：换视角或换场景
副歌（再爆发）：最后一次，最强烈
尾奏：留一个意象结束
```

**五感分配策略**：

- 主歌：视觉为主 + 1个辅助感官（听觉/嗅觉）
- 预副歌：听觉渐强（暗示情绪升级）
- 副歌：综合感官冲击
- 桥段：触觉为主（回归身体感受）
- 尾奏：单一视觉意象（余韵）

### Phase 6: 输出

输出两份内容：

#### 歌词正文

```markdown
# [歌曲标题]

## 主歌A
[视觉场景描写 + 1个听觉/嗅觉细节]

## 主歌B
[物件/动作叙事]

## 预副歌
[情绪转折]

## 副歌
[核心 Hook × 3]
[画面余韵]

## 桥段
[换视角/换场景]

## 副歌
[重复]

## 尾奏
[单一意象]
```

#### Suno/Udio Prompt

```markdown
---
歌词汤底：
[完整歌词，标注段落标记]

旋律汤底：
风格：[风格]
BPM：[BPM]
调式：[调式]
结构：[对应结构]
乐器编配：
  前奏：[乐器]
  主歌：[乐器]
  副歌：[乐器]
  桥段：[乐器]
情感基调：[情感]
人声类型：[人声]
---

以上是参考约束。请基于歌词生成完整歌曲。
```

如果用户指定了目标工具不是 Suno，按对应工具的 prompt 格式适配：
- **豆包**：中文 prompt 更友好，结构相同
- **Udio**：英文 prompt 更佳，需翻译歌词意境描述

以下是场景辞海中部分场景与情感的对应关系，用于辅助 Phase 2 的检索：

### 失恋/遗憾
- 厨房、卧室、深夜便利店、雨中街道、空荡的房间、旧物
- 常见五感：冷（触觉）、暗（视觉）、静（听觉）、旧味道（嗅觉）

### 思念
- 老车站、港口、旧书店、秋天的公园、咖啡馆角落
- 常见五感：远（视觉）、回声（听觉）、旧纸味（嗅觉）、微凉（触觉）

### 励志/热血
- 天台、山顶、海边日出、运动场、空旷公路
- 常见五感：亮（视觉）、风声（听觉）、热（触觉）、开阔（视觉）

### 甜蜜/温暖
- 厨房、花园、咖啡馆、书店、沙发
- 常见五感：暖光（视觉）、笑声（听觉）、甜香（嗅觉）、柔软（触觉）

### 孤独/疏离
- 深夜便利店、空旷停车场、雨中公交站、凌晨的街
- 常见五感：白光刺眼（视觉）、嗡嗡声（听觉）、消毒水味（嗅觉）、冷（触觉）

**注意**：以上仅为辅助参考，实际检索应以 `search_notes` 的结果为准。场景辞海包含 222 个场景，远超以上列举。

## 质量规则

1. **每段歌词至少 1 个具体物象**：不允许整段只有抽象情感词
2. **副歌 Hook ≤ 9 字**：短句才容易被记住
3. **重复核心句 ≥ 2 次**：重复是洗脑的基础
4. **反套路检查通过**：不得出现常见陷阱中的陈词滥调
5. **五感覆盖 ≥ 3 种**：纯视觉歌词缺少穿透力
6. **韵脚统一**：每段的韵脚应保持一致
7. **不引入场景中没有的感官**：素材必须来自检索到的场景，不得凭空编造

## 保存到知识库

当用户说"保存歌词"或"加入我的作品"时：

1. 用 `create_note` 保存，标题格式：`🎵 [风格] - [主题] - 歌词`
2. 内容包含：歌词正文 + Suno prompt + 使用的场景素材来源
3. 可选加入用户指定的集合
